## Before i setup this folder:

Notes: 
ISSUE:
static html files cant protect themselves so we cant just have everything be html someone could just type website/dashboard and bypass login
FIX:
express needs to act as a gatekeeper using Middleware. when a user request /dashboard node should check thier security token (session or JWT). if they are authed then it /dashboard goes through if not they redirect to login page. 

ISSUE:
admin and normal users cant share general access routes. account compromise could lead to full database escalation.
FIX: 
Role-Based Access Control(RBAC). in SQLite every user needs a 'role' field (role:operator, role: admin)  the admin backend routes must strictly check for role === 'admin'. also shroud the admin path. instead of /admin do something
like /api/v1/system-override-control or something even more discrete.

ISSUE: 
automated system messages (Race Conditions). if admin triggers a message to 500 users at once inside a standard HTTP request loop the node server will lock up trying to proceessa ll the database writes at once and performance will drop to zero
for anyone else trying to use the app. basically Denial of Service
FIX:
for mass messaging and automated emails look into an asynchronous background event handler or a  lightweight queue. for my homelab scale i will just ensure my DB writes using non-blocking asynchronous operations (async/wait) so the main 
thread stays open to accept other incoming traffic
